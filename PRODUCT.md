# Product

## Register

product

## Users

NeekShot is for Mac users who need screenshots while staying in flow: developers debugging UI, documenting bugs, saving transient app states, or capturing evidence while another task is still happening.

## Product Purpose

NeekShot is a local-first macOS menu bar utility for fast, silent screenshots. Its first job is to capture the current main display from a global hotkey, save it locally, copy it to the clipboard, and avoid opening an overlay, editor, picker, or focus-stealing window.

Success means a user can press one shortcut and keep typing, dragging, watching, or debugging while the capture completes in the background.

## Brand Personality

Quiet, precise, utilitarian.

The app should feel like a native Mac utility that disappears into the workflow. It should be clear about permissions and state, but avoid marketing-style UI, decoration, or cleverness.

## Anti-references

Avoid becoming a full screenshot editor, cloud sharing product, social screenshot beautifier, or annotation suite. CleanShot X, Shottr, Snagit, and Xnapper are strong tools, but NeekShot should not compete on editor depth or presentation polish first.

Avoid UI that interrupts the user after every capture. Avoid automatic uploads. Avoid persistent history until users explicitly choose it.

## Design Principles

1. Preserve flow before adding modes.
2. Keep capture local unless the user explicitly opts into sharing.
3. Prefer one reliable silent path over many partially working capture modes.
4. Explain macOS permissions plainly and only when needed.
5. Use native controls and system behavior wherever possible.

## Accessibility & Inclusion

Use native macOS controls, labels, menu items, and keyboard shortcuts. Support reduced motion by avoiding nonessential animation. Keep status feedback available through menu item text and notifications only when explicitly enabled later.
